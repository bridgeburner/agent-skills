# Slide Template

Complete HTML template and base CSS for viewport-fitting slide presentations. Every slide deck MUST include these styles and the SlidePresentation controller.

## HTML Shell

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Presentation Title</title>

    <!-- Fonts: choose appropriate display + body pair -->
    <link rel="stylesheet" href="FONT_URL_HERE">

    <style>
        /* === THEME CUSTOM PROPERTIES === */
        :root {
            /* Colors — set these per deck */
            --bg-primary: #0a0f1c;
            --bg-secondary: #111827;
            --text-primary: #ffffff;
            --text-secondary: #9ca3af;
            --accent: #00ffcc;
            --accent-secondary: var(--accent);
            --accent-glow: rgba(0, 255, 204, 0.3);

            /* Typography — MUST use clamp() */
            --font-display: 'Clash Display', sans-serif;
            --font-body: 'Satoshi', sans-serif;
            --title-size: clamp(2rem, 5vw, 4rem);
            --h2-size: clamp(1.5rem, 3.5vw, 2.75rem);
            --h3-size: clamp(1.25rem, 2.5vw, 2rem);
            --body-size: clamp(1rem, 2vw, 1.5rem);
            --small-size: clamp(0.875rem, 1.5vw, 1.125rem);

            /* Spacing — MUST use clamp() */
            --slide-padding: clamp(1rem, 4vw, 4rem);
            --content-gap: clamp(0.5rem, 2vw, 2rem);
            --element-gap: clamp(0.25rem, 1vw, 1rem);

            /* Animation */
            --ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);
            --ease-out-back: cubic-bezier(0.34, 1.56, 0.64, 1);
            --ease-in-out-smooth: cubic-bezier(0.65, 0, 0.35, 1);
            --duration-fast: 300ms;
            --duration-normal: 600ms;
            --duration-slow: 1000ms;
            --stagger-step: 80ms;
        }

        /* === BASE CSS (see sections below) === */
    </style>
</head>
<body>
    <!-- Progress bar -->
    <div class="progress-bar"></div>

    <!-- Navigation dots -->
    <nav class="nav-dots">
        <!-- Generated by JS -->
    </nav>

    <!-- Slides -->

    <!-- Title slide with hero background image -->
    <section class="slide title-slide">
        <div class="slide-bg" style="background-image: url('{name}-assets/slide-01-hero.png')"></div>
        <div class="slide-content">
            <h1 class="reveal">Presentation Title</h1>
            <p class="reveal">Subtitle or author</p>
        </div>
    </section>

    <!-- Content slide with inset image -->
    <section class="slide">
        <div class="slide-content">
            <h2 class="reveal">Slide Title</h2>
            <p class="reveal">Content...</p>
            <figure class="slide-figure reveal">
                <img class="slide-image" src="{name}-assets/slide-03-concept.png" alt="Descriptive alt text">
                <figcaption>Optional caption</figcaption>
            </figure>
        </div>
    </section>

    <!-- Text-only slide -->
    <section class="slide">
        <div class="slide-content">
            <h2 class="reveal">Slide Title</h2>
            <p class="reveal">Content...</p>
        </div>
    </section>

    <!-- More slides... -->

    <script>
        /* SlidePresentation controller (see section below) */
    </script>
</body>
</html>
```

## Base CSS

All styles below are mandatory in every deck. The `<style>` block in the HTML shell above contains the `:root` custom properties; the rules below follow immediately after them.

### Reset

```css
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
```

### Z-Index Stacking Order

Reference for layering decisions. Not a CSS rule itself.

| Layer | z-index | Purpose |
|-------|---------|---------|
| `.slide-bg` | 0 | Background images |
| `.slide-content` (after `.slide-bg`) | 1 | Main content over backgrounds |
| Decorative elements | 2 | Section numbers, watermarks, floating accents |
| Slide navigation | 3 | Progress bar, nav dots |
| Overlays | 4 | Modals, expanded images, lightboxes |

### Viewport Fitting (mandatory)

These rules lock every slide to exactly one viewport. They MUST be present.

```css
/* 1. Lock html/body to viewport */
html, body {
    height: 100%;
    overflow-x: hidden;
}

html {
    scroll-snap-type: y mandatory;
    scroll-behavior: smooth;
}

body {
    font-family: var(--font-body);
    background: var(--bg-primary);
    color: var(--text-primary);
}

/* 2. Each slide = exact viewport height */
.slide {
    width: 100vw;
    height: 100vh;
    height: 100dvh; /* Dynamic viewport height for mobile browsers */
    overflow: hidden; /* CRITICAL: Prevent ANY overflow */
    scroll-snap-align: start;
    display: flex;
    flex-direction: column;
    position: relative;
}

/* 3. Content container with flex for centering */
.slide-content {
    flex: 1;
    display: flex;
    flex-direction: column;
    justify-content: center;
    max-height: 100%;
    overflow: hidden; /* Double-protection against overflow */
    padding: var(--slide-padding);
}
```

### Typography Scale

Defined in `:root` via the custom properties block. All typography MUST use `clamp()` for responsive scaling. See the Theme Custom Properties table for the full list.

### Spacing Scale

Also defined in `:root`. The three spacing tokens are:

- `--slide-padding` -- outer padding on `.slide-content`
- `--content-gap` -- gap between major content blocks
- `--element-gap` -- gap between fine-grained elements (list items, icon+label, etc.)

### Content Containers

```css
/* 5. Cards/containers use viewport-relative max sizes */
.card, .container, .content-box {
    max-width: min(90vw, 1000px);
    max-height: min(80vh, 700px);
}

/* 6. Lists auto-scale with viewport */
.feature-list, .bullet-list {
    gap: clamp(0.4rem, 1vh, 1rem);
}

.feature-list li, .bullet-list li {
    font-size: var(--body-size);
    line-height: 1.4;
}

/* 7. Grids adapt to available space */
.grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(min(100%, 250px), 1fr));
    gap: clamp(0.5rem, 1.5vw, 1rem);
}

/* 8. Images constrained to viewport */
img, .image-container {
    max-width: 100%;
    max-height: min(50vh, 400px);
    object-fit: contain;
}
```

### Basic Reveal

The one animation included in base CSS. All other animations belong in a separate reference.


## SlidePresentation Controller

Complete JavaScript class. Place in a `<script>` tag at the end of `<body>`.

```javascript
class SlidePresentation {
    constructor() {
        this.slides = [...document.querySelectorAll('.slide')];
        this.currentIndex = 0;
        this.progressBar = document.querySelector('.progress-bar');
        this.navContainer = document.querySelector('.nav-dots');
        this.reducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
        this.touchStartY = 0;

        this.buildNavDots();
        this.setupIntersectionObserver();
        this.setupKeyboard();
        this.setupTouch();
        this.updateProgress();
    }

    /* --- Nav dots --- */
    buildNavDots() {
        if (!this.navContainer) return;
        this.navContainer.innerHTML = this.slides
            .map((_, i) => `<button class="nav-dot${i === 0 ? ' active' : ''}" aria-label="Go to slide ${i + 1}"></button>`)
            .join('');
        this.dots = [...this.navContainer.querySelectorAll('.nav-dot')];
        this.navContainer.addEventListener('click', e => {
            const dot = e.target.closest('.nav-dot');
            if (dot) this.goTo(this.dots.indexOf(dot));
        });
    }

    /* --- IntersectionObserver: track current slide + trigger .visible --- */
    setupIntersectionObserver() {
        const observer = new IntersectionObserver(entries => {
            entries.forEach(entry => {
                if (entry.isIntersecting) {
                    entry.target.classList.add('visible');
                    const idx = this.slides.indexOf(entry.target);
                    if (idx !== -1) {
                        this.currentIndex = idx;
                        this.updateProgress();
                        this.updateDots();
                    }
                }
            });
        }, { threshold: 0.5 });
        this.slides.forEach(s => observer.observe(s));
    }

    /* --- Keyboard navigation --- */
    setupKeyboard() {
        document.addEventListener('keydown', e => {
            switch (e.key) {
                case 'ArrowDown': case 'ArrowRight': case ' ': case 'PageDown':
                    e.preventDefault(); this.next(); break;
                case 'ArrowUp': case 'ArrowLeft': case 'PageUp':
                    e.preventDefault(); this.prev(); break;
                case 'Home':
                    e.preventDefault(); this.goTo(0); break;
                case 'End':
                    e.preventDefault(); this.goTo(this.slides.length - 1); break;
            }
        });
    }

    /* --- Touch / swipe support --- */
    setupTouch() {
        document.addEventListener('touchstart', e => {
            this.touchStartY = e.touches[0].clientY;
        }, { passive: true });
        document.addEventListener('touchend', e => {
            const dy = this.touchStartY - e.changedTouches[0].clientY;
            if (Math.abs(dy) > 50) dy > 0 ? this.next() : this.prev();
        });
    }

    /* --- Navigation helpers --- */
    next() { this.goTo(Math.min(this.currentIndex + 1, this.slides.length - 1)); }
    prev() { this.goTo(Math.max(this.currentIndex - 1, 0)); }
    goTo(index) {
        this.slides[index].scrollIntoView({ behavior: this.reducedMotion ? 'auto' : 'smooth' });
    }

    /* --- UI updates --- */
    updateProgress() {
        if (!this.progressBar) return;
        const pct = ((this.currentIndex + 1) / this.slides.length) * 100;
        this.progressBar.style.width = pct + '%';
    }
    updateDots() {
        if (!this.dots) return;
        this.dots.forEach((d, i) => d.classList.toggle('active', i === this.currentIndex));
    }
}

// Initialize when DOM is ready
new SlidePresentation();
```

## Theme Custom Properties

Quick-reference list of all CSS custom properties that control the deck's look. Change these to retheme the entire presentation.

### Colors

| Property | Controls | Default |
|----------|----------|---------|
| `--bg-primary` | Page/slide background | `#0a0f1c` |
| `--bg-secondary` | Alternate background (cards, panels) | `#111827` |
| `--text-primary` | Headings and body text | `#ffffff` |
| `--text-secondary` | Captions, muted text | `#9ca3af` |
| `--accent` | Primary accent (links, highlights, gradient start) | `#00ffcc` |
| `--accent-secondary` | Secondary accent (gradient end); falls back to `--accent` | `var(--accent)` |
| `--accent-glow` | Glow/shadow color derived from accent | `rgba(0, 255, 204, 0.3)` |

### Typography

| Property | Controls | Default |
|----------|----------|---------|
| `--font-display` | Headings and titles | `'Clash Display', sans-serif` |
| `--font-body` | Body text, lists, captions | `'Satoshi', sans-serif` |
| `--title-size` | `<h1>` font size | `clamp(2rem, 5vw, 4rem)` |
| `--h2-size` | `<h2>` font size | `clamp(1.5rem, 3.5vw, 2.75rem)` |
| `--h3-size` | `<h3>` font size | `clamp(1.25rem, 2.5vw, 2rem)` |
| `--body-size` | Paragraph and list item font size | `clamp(1rem, 2vw, 1.5rem)` |
| `--small-size` | Captions, fine print | `clamp(0.875rem, 1.5vw, 1.125rem)` |

### Spacing

| Property | Controls | Default |
|----------|----------|---------|
| `--slide-padding` | Outer padding on `.slide-content` | `clamp(1rem, 4vw, 4rem)` |
| `--content-gap` | Gap between major content blocks | `clamp(0.5rem, 2vw, 2rem)` |
| `--element-gap` | Gap between fine-grained elements | `clamp(0.25rem, 1vw, 1rem)` |

### Animation

| Property | Controls | Default |
|----------|----------|---------|
| `--ease-out-expo` | Primary easing curve | `cubic-bezier(0.16, 1, 0.3, 1)` |
| `--ease-out-back` | Easing with slight overshoot | `cubic-bezier(0.34, 1.56, 0.64, 1)` |
| `--ease-in-out-smooth` | Symmetric ease for loops | `cubic-bezier(0.65, 0, 0.35, 1)` |
| `--duration-fast` | Quick transitions | `300ms` |
| `--duration-normal` | Standard reveal/fade | `600ms` |
| `--duration-slow` | Dramatic entrance | `1000ms` |
| `--stagger-step` | Delay increment between staggered children | `80ms` |

## Common Assembly Errors

These four errors account for most slide deck failures when assembling from the reference files. Check each one before finalizing a deck.

1. **Missing shared split/image styles.** Split layouts (`split-50-50`, `split-60-40`, `split-40-60`) require the shared `[class*="split-"]` base styles AND image sizing overrides from slide-layouts.md. These live in separate "Shared Styles" sections that apply to ALL split and image-forward layouts. Include them or columns will not render correctly.

2. **Wrong animation wrapping.** Entrance animations use a reduced-motion-first pattern. Place the base `opacity: 0` and `transform` declarations OUTSIDE `@media (prefers-reduced-motion: no-preference)`, and the `@keyframes` plus `.visible` animation assignments INSIDE it. Putting everything outside ignores user preference; putting everything inside leaves elements permanently invisible when reduced motion is active.

3. **Competing IntersectionObservers.** `SlidePresentation` (this file) creates an observer that adds `.visible` to slides. `initSlideAnimations()` (slide-animations.md) creates a second observer for animation triggering. When assembling, either merge them into one observer or ensure both run without conflict. Let SlidePresentation own `.visible` toggling; let initSlideAnimations handle animation-specific class application.

4. **Missing required HTML elements.** The `SlidePresentation` controller expects `<div class="progress-bar"></div>` and `<nav class="nav-dots"></nav>` in the DOM. Omitting them does not cause errors — the controller guards against null — but the progress indicator and dot navigation silently break. Always include both elements.
